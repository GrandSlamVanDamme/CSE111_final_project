2. Objective Definition

We need to reliably predict the temperature of the reactor (in degrees Centigrade) from the time passed since shutoff (in hours). Since this is a live nuclear reactor, criticality is a prime safety concern.

3. Assumptions and Simplifications

In any mathematical model, there are certain assumptions made and simplifications accepted. For this model, we keep in mind the sensitive nature of the system in question, which leades us to an overarching principle which we’ll follow throughout this modelling process.

Our first and central assumption is that of the worst-case scenario: we assume the least optimal conditions, in every case, for time to cooling. For example, when we assume a constant external temperature, we assume that temperature to be the mean daily maximum for the season in which we operate. We assume high humidity that will slow the cooling rate for the reactor components, etc.

  1. Bulk, uniform cooling rate: no regions of greater \(T\) density
  2. Single substance component: single specific heat
  3. Constant external temperature
  4. Constant external humidity, pressure: We assume no weather.
  5. We assume the absence of other heat sources, ignoring the effect of sunlight.
  6. Reactor isn’t operated until cooling is complete
  7. We assume a \(-5{\%}\) margin of error with temperature measurement (The observed temeperatures have been increased by a factor of \(1.05\)). This will be one in a group of safety redundancies.

4. Variables and Parameters

\(T_R\) is the reactor temperature.

\(T_E\) is the external temperature

\(t\) is the time from shutdown in hours.

That’s it! While a high-fidelity model would and could account for the specific heats of individual reactor components, the role of humidity in cooling, and the stochastic nature of thermodynamic processes, we can simplify extensively for the purpose of this problem. Furthermore, we’re not supplied with external temperature data. Therefore, we’ll leave out the external temperature as an explicit variable and we’ll model the cooling rate exclusively in terms of the reactor temperature and time passed.

First, we’re supplied with cooling and time data; this means that we must assume similar external conditions throughout the cooling period in order to use this supplied data for interpolation and limited extrapolation. Second, we can justify some simplification from the safety-focused nature of the model we’re designing. We do not know the external conditions for the data we’re given, so the most prudent approach is to assume that these conditions are favorable to quick cooling and consider them optimized. We will start, at least, from a basic model of heat transfer. This is most appropriate for the limited data we have on the situation.

def leastsquerror(Y, F):
    """
    Takes two lists, Y = [data points] and F(x) = [func_values].
    Computes cumulative variances of a series of residuals
    returns list of variances
    """
    resid = residuals(Y, F)

    squares = []

    squm = []

    for r in resid:
        squares.append(r**2)

        squm.append(sum(squares))

    return squm


def absdev_error(Y, F):
    """
    Takes two lists, Y = [data points] and F(x) = [func_values].
    computes cumulative absolute deviation
    returns list of absolute deviations
    """

    resid = residuals(Y, F)

    normals = []

    absum = []

    for r in resid:
        normals.append(abs(r))

        absum.append(sum(normals))

    return absum


def cheb_error(Y, F):
    """
    Takes two lists, Y = [data points] and F(x) = [func_values].
    returns maximum residual
    """

    cheb = abs(max(residuals(Y, F)))

    return cheb


def residuals(Y, F, func_type="linear"):
    """
    Takes two lists, Y = [data points] and F(x) = [func_values]
    computes their residuals
    returns list of residuals

    """

    resid = []
    if func_type == "exponential" or func_type == "power":
        Y = [np.e**y for y in Y]
    else:
        pass

    for f, y in zip(F, Y):
        resid.append(y - f)

    return resid

def objectivist(X, Y, n):
    """
    Takes two arrays, x = [data 1] and Y = [data 2].
    f(x) = c_n*x^n + c_(n-1)x^(n-1)+...c_0x^0 with n parameters
    returns objective function f
    """

    mat_x = []
    mat_y = []
    x_list = []

    for Xi, Yi in zip(X, Y):
        i = n

        rightrow = []
        leftrow = []
        objective = []
        bounds = []

        while i > -1:
            bounds.append((None, None))
            objective.append(0)
            rightrow.append(Xi**i)
            leftrow.append(-(Xi**i))
            x_list.append(Xi**i)

            i -= 1

        rightrow.append(-1)
        leftrow.append(-1)
        mat_x.append(rightrow)
        mat_x.append(leftrow)
        mat_y.append(Yi)
        mat_y.append(-Yi)

    bounds.append((0, None))
    objective.append(1)

    barry = [objective, mat_x, mat_y, bounds, x_list]
    return barry

def sum_maker(X, Y, n):
    """
    Takes two arrays, x = [data 1] and Y = [data 2]
    and desired polynomial degree n. Returns a matrix of
    sums of x values and a matrix of sums of y values
    """

    i = n

    sum_list_you_got_there = []
    neg_sum_list = []

    while i > -1:
        power_list = []
        neg_pow_list = []

        for Xi in X:
            it = Xi**i
            power_list.append(it)
            neg_pow_list.append(-it)

        sum_list_you_got_there.append(sum(power_list))
        neg_sum_list.append(sum(neg_pow_list))

        i -= 1

    x_mat = [sum_list_you_got_there, neg_sum_list]
    y_mat = [sum(Y), -sum(Y)]

    return [x_mat, y_mat]


def absdev_fit(X, Y, n):
    """
    takes X data and makes a list of powers
    (X^n, X^n-1...),
    dots it with undefined params to whip up
    a cost function that absdev_fit
    will minimize

    #####################################################################################################################
    ##  https://stackoverflow.com/questions/51883058/l1-norm-instead-of-l2-norm-for-cost-function-in-regression-model  ##
    #####################################################################################################################

    used as guideline and partial template
    """

    def cost_func(params):
        # sympy removal courtesy of gemini thread listed up top
        # SciPy will plug the current numerical guess into 'params'
        f = np.polyval(params, X)
        # Return the L1 norm (sum of absolute deviations)
        return np.sum(np.abs(Y - f))

    """
    f = np.polyval(params, X)
    cost_func = np.sum(np.abs(Y - f))
    """

    guess = (n + 1) * [1.0]

    result = sci.optimize.minimize(cost_func, guess, method="Nelder-Mead")

    return result.x


def LS2_fit(X, Y, n):
    """
    Takes two arrays, x = [data 1] and Y = [data 2].
    fits model function
    f(x) = c_n*x^n + c_(n-1)x^(n-1)+...c_0x^0 with n parameters
    such that (sum|Yi-Fi|^2, i ϵ NN) is minimized.
    returns f(x) as an array
    """
    LS2 = np.polyfit(X, Y, n)

    return LS2


def x_list(X):
    """
    takes X data, returns linspaced list of X-powers
    from X^n to X^0.
    """
    X = np.linspace(X[0], X[-1] + np.mean(X) / 1000, 1000)

    return X

Exponential decay:

Of form \(T(t) = (c_2)e^{c_1t}\) (\(c_2\) is negative).

We’ll linearize this for two reasons; so that this fit can be optimized with linear programming and for the sake of visual clarity. Taking the natural log of both sides gives us

def leastsquerror(Y, F):
    """
    Takes two lists, Y = [data points] and F(x) = [func_values].
    Computes cumulative variances of a series of residuals
    returns list of variances
    """
    resid = residuals(Y, F)

    squares = []

    squm = []

    for r in resid:
        squares.append(r**2)

        squm.append(sum(squares))

    return squm


def absdev_error(Y, F):
    """
    Takes two lists, Y = [data points] and F(x) = [func_values].
    computes cumulative absolute deviation
    returns list of absolute deviations
    """

    resid = residuals(Y, F)

    normals = []

    absum = []

    for r in resid:
        normals.append(abs(r))

        absum.append(sum(normals))

    return absum


def cheb_error(Y, F):
    """
    Takes two lists, Y = [data points] and F(x) = [func_values].
    returns maximum residual
    """

    cheb = abs(max(residuals(Y, F)))

    return cheb


def residuals(Y, F, func_type="linear"):
    """
    Takes two lists, Y = [data points] and F(x) = [func_values]

    returns list of residuals

    """

    resid = []
    if func_type == "exponential" or func_type == "power":
        Y = [np.e**y for y in Y]
    else:

    for f, y in zip(F, Y):
        resid.append(y - f)

    return resid

def sum_maker(X, Y, n):
    """
    Takes two arrays, x = [data 1] and Y = [data 2]
    and desired polynomial degree n. Returns a matrix of
    sums of x values and a matrix of sums of y values
    """

    i = n

    sum_list_you_got_there = []
    neg_sum_list = []

    while i > -1:
        power_list = []
        neg_pow_list = []

        for Xi in X:
            it = Xi**i
            power_list.append(it)
            neg_pow_list.append(-it)

        sum_list_you_got_there.append(sum(power_list))
        neg_sum_list.append(sum(neg_pow_list))

        i -= 1

    x_mat = [sum_list_you_got_there, neg_sum_list]
    y_mat = [sum(Y), -sum(Y)]

    return [x_mat, y_mat]

7. Validation

We chose a Chebyshev error-optimized exponential fit to model the cooling rate of this nuclear plant reactor. This is because the worst case scenario (the largest single residual) is minimized through Chebyshev optimization. We need to account not just for a comprehensive overview of error, but for the Murphy’s Law example where the air is hot and humid, the cooling pool circulator is broken, and all other systems operate at minimal effectiveness. We must work under these assumptions because an overly optimistic prediction could have unacceptable consequences to the plant and its environs. We chose the exponential fit because it minimized the Chebyshev error most effectively and did so with the least over-fitting of any model. This model for the cooling rate is backed up empirically. It’s a barely-modified form of Newton’s Law of cooling: \[ T(t) = T_R + (T_E - T_R)e^{-kt} \] We are only concerned with the medium, that is the reactor itself, so we don’t include the \(T(t)\) term, instead letting the reactor temperature take its place.

\[\begin{aligned} T_R(t) & = (T_E - T_R)e^{-kt} \\ \ln(T_R(t)) & = \ln[(T_E-T_R)e^{-kt}] \\ \ln(T_R(t)) & = \ln(T_E - T_R) & - kt \\ \ln(T_R(t)) & = c_1 & -c_0t \\ \end{aligned}\]

Where \(T_R\) is the reactor temperature and \(T_E\) is the external temperature, substituted for \(T_0\) due to our assumption that the external/surrounding air temperature is constant across the range of observation.

8. Re-evaluation and Improvement

This model has a major weakness in that it approaches the problem deterministically. In reality, we would benefit from introducing randomness into our analysis, since that could provide closer estimates of daily temperature, humidity, and component failure rates while retaining this model’s emphasis on multi-system error. That would enable a more efficient response to uncertainty than the chosen model where all predictions are made according to an edge case.

Additionally, we assumed bulk cooling, here. Further consideration of this problem as a stochastic process would involve the treatment of heat density within the reactor. This would allow for a more accurate thermodynamical analysis and enable response to uneven coooling rates; we should be equally concerned with a single region of retained heat as with a higher temperature across the bulk. This more accurate treatment would involve solving the PDE version of the heat equation

\[ \frac{\partial u}{\partial t} = k \nabla^2 u + \frac{Q(x, t)}{c\rho} \]

And using what data we have to narrow down the resultant Fourier series family of solutions into a range of particular solutions. For the scope of the problem as presented, a simpler solution is valid as long as it overcorrects enough to preserve safety requirements.
